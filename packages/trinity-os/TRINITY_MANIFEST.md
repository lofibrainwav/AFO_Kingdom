# TRINITY-OS 매니페스트

## 시스템 정의

TRINITY-OS는 AFO 왕국의 통합 자동화 운영체제입니다.

## 구성 요소

### Core Scripts
- `kingdom_problem_detector.py` - 문제 감지 엔진
- `kingdom_auto_recovery.py` - 자동 복구 시스템
- `kingdom_spirit_integration.py` - 왕국 정신 통합
- `kingdom_health_report.py` - 건강 상태 리포트

### Automation Scripts
- `kingdom_unified_autorun.sh` - 통합 자동화 실행
- `kingdom_infinite_autorun.sh` - 끝까지 오토런
- `test_unified_autorun.sh` - 통합 테스트
- `verify_all_scripts.sh` - 검증 스크립트

### Configuration
- `.vscode/settings.json` - VSCode 설정
- `.cursor/environment.json` - Cursor 환경 설정
- `.cursorrules` - Cursor 규칙
- `requirements.txt` - Python 의존성

### Documentation
- `README.md` - 메인 문서
- `KINGDOM_UNIFIED_AUTORUN_GUIDE.md` - 사용 가이드
- `CURSOR_REVIEW_DISABLE_GUIDE.md` - Cursor 설정 가이드
- `TRINITY_MANIFEST.md` - 이 파일

### Personas (SSOT)
- `TRINITY_OS_PERSONAS.yaml` - 웹/CLI 페르소나·명칭·역할 정본

## 철학

**眞善美孝永 (Truth, Goodness, Beauty, Serenity, Eternity)**

### 眞 (Truth)
- 정확한 문제 감지
- 진실된 데이터 기반 운영
- NO MOCK, NO HARDCODING

### 善 (Goodness)
- 인간 중심의 윤리적 자동화
- 지속 가능한 시스템 운영

### 美 (Beauty)
- 단순하고 우아한 인터페이스
- 사용자의 인지 부하 최소화

### 孝 (Serenity)
- 형님의 평온을 최우선
- 마찰 제거와 효율성 극대화

### 永 (Eternity)
- 지속 가능한 영속성 보장
- 시스템의 장기적 안정성

## 실행 방식

### Interactive Mode
```bash
./run_trinity_os.sh
```

### Direct Mode
```bash
# 문제 감지
python3 scripts/kingdom_problem_detector.py

# 건강 리포트
python3 scripts/kingdom_health_report.py

# 통합 자동화
./scripts/kingdom_unified_autorun.sh

# 끝까지 오토런
./scripts/kingdom_infinite_autorun.sh
```
